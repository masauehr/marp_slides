# データ収集の課題と解決

---

# 課題: 発電データがCSVで取れない

スマホアプリは**棒グラフ表示のみ** — データエクスポート機能なし

```
┌─────────────────────────────┐
│  ソーラー発電アプリ          │
│                             │
│  █                         │
│  █  █                      │  ← 棒グラフが表示されるだけ
│  █  █  █  █                │     数値をコピーする手段がない
│  ─────────────────          │
│  月  火  水  木  金          │
└─────────────────────────────┘
```

**解決策**: スクリーンショットを AI に送り、棒グラフの数値を自動読み取り

---

# AI による棒グラフ自動読み取り

スクリーンショット → Claude（AI）へ送信 → 数値を CSV に変換・蓄積

```
スマホ画面 ──→ AI（Claude）──→ CSV蓄積
  screenshot   「棒グラフから    date, generation_kwh
                発電量を読み取   2025-06-01, 18.4
                ってCSV形式で    2025-06-02, 14.2
                出力して」       2025-06-03, 21.7
                                 ...（351日分）
```

**実績**: 2025年6月〜2026年6月（351日）の発電量データを構築  
AI読み取りの誤差は目視確認で修正 → **実用精度のデータセットが完成**

> デジタルデータを持たないレガシーなシステムでも、AIを介することで機械学習の入力データに変換できる

---

# AMD 農研機構メッシュ気象データ

**農研機構（NARO）が提供する 1km メッシュ気象データ**

| データ種別 | 変数 | 説明 |
|-----------|------|------|
| 予報値（将来日） | `GSR` | 全天日射量（MJ/m²/日） |
| 予報値（将来日） | `SSD` | 日照時間（h/日） |
| 実況値（確定値） | `GSR` | 前日までの実測気象値 |

**活用のポイント**
- 予報値は **翌日以降の値を API で取得可能** → 前日予測に使える
- 実況値は **前日の確定値** → 予測精度の事後検証に使える
- 平年値（気候値）との比率（`GSR_ratio`）で季節性を標準化

> 自宅 GPS 座標を指定するだけで、1km メッシュの地点値を自動取得

# 機械学習モデルの構築

---

# 説明変数（v3）

| 変数名 | 説明 | 取得元 |
|--------|------|--------|
| `GSR` | 全天日射量（MJ/m²/日） | AMD 予報 |
| `GSR_ratio` | GSR ÷ 平年値（季節性を標準化） | AMD 予報 ÷ AMD 気候値 |
| `SSD` | 日照時間（h/日） | AMD 予報 |

**変数選定の経緯**

- 気温・GSR5日平均・月変数など8変数から開始（v1）
- 相関分析・Permutation Importance で絞り込み → 3変数（v3）で精度向上
- `GSR_ratio` が最重要変数：同じ日射量でも季節による発電効率の差を吸収

> 変数を減らすことで過学習が抑制され、CV MAE: 2.23 → 1.89 kWh に改善

---

# 交差検証（TimeSeriesSplit）

時系列を崩さずに5分割し「過去で学習 → 直後で検証」を繰り返す

```
Fold 1: [学習────────] 検証: 2025/08–09  夏〜秋
Fold 2: [学習─────────────] 検証: 2025/09–11  秋
Fold 3: [学習──────────────────] 検証: 2025/11–2026/01  冬
Fold 4: [学習─────────────────────────] 検証: 2026/01–03  冬〜春
Fold 5: [学習────────────────────────────────] 検証: 2026/03–06  春〜梅雨
```

各 fold の予測値（OOF）を連結 → **過学習のない公正な精度評価**

| 指標 | 値 |
|------|----|
| R²（OOF） | 0.846 |
| MAE（OOF） | **1.89 kWh/日** |

# 日次自動運用と継続検証

---

# 毎日の自動実行フロー

**毎朝 6:30 に launchd（macOS）が自動起動**

```
6:30 起動
  │
  ├─① AMD API から当日の GSR・SSD 予報を取得
  │     （自宅座標 N26.17, E127.74 の 1km メッシュ値）
  │
  ├─② RF モデルで当日発電量を予測
  │     v3（GSR・GSR_ratio・SSD）と v3b（SSD なし）を並行予測
  │
  ├─③ 面的分布図を生成・蓄積
  │     spatial_fcst_YYYYMMDD.png（予報）
  │     spatial_actual_YYYYMMDD.png（前日実況）
  │
  ├─④ README.md を自動更新
  │     予測値・実測値・誤差・グラフを上書き
  │
  └─⑤ GitHub に自動 push
        → Web 上で誰でも最新の予測・検証結果を確認可能
```

# AMD 予報精度の検証（GSR・SSD）

AMD の予報が実際にどのくらい当たるかを**毎日継続的に検証**

| 変数 | 翌日予報 MAE | 特記事項 |
|------|------------|---------|
| GSR（全天日射量） | 約 3〜4 MJ/m²/日 | 概ね良好 |
| SSD（日照時間） | 約 7〜8 h/日 | 晴れ日に著しく過小評価 |

> 予報精度の継続検証により、SSD 過小評価がモデル誤差の原因と特定できた

# 3モデル比較結果

---

# 全体精度の比較

同一説明変数（GSR・GSR_ratio・SSD）で3モデルを比較

| モデル | R²（OOF） | *MAE* | 特徴 |
|--------|-----------|-------|------|
| **Random Forest v3** | **0.846** | ***1.89 kWh*** | 全体最良・現行モデル |
| Gradient Boosting | 0.840 | 1.93 kWh | RF とほぼ同等 |
| Ridge 回帰（線形） | 0.813 | 2.14 kWh | 全体は劣るが高発電日に特性あり |

- GB と RF はどちらも「アンサンブル平均」→ 性能が似た理由
- Ridge は線形なので高日射日に異なる挙動を示す

---

# 高発電日の比較（実測 ≥ 19 kWh, n=77）

| モデル | 高発電日 MAE | 全体 MAE | 差 |
|--------|------------|---------|-----|
| Random Forest v3 | 2.55 kWh | 1.89 kWh | +0.66 |
| Gradient Boosting | 2.31 kWh | 1.93 kWh | +0.38 |
| **Ridge 回帰** | **2.28 kWh** | 2.14 kWh | +0.14 |

> 線形モデル（Ridge）は**高発電日の過小予測が最も小さい**  
> RF/GB はアンサンブル平均により高値が中央値に引き寄せられる（平均回帰）

---

# fold 別 MAE（季節性の分析）

| Fold | 検証期間 | 季節 | RF v3 | GB | Ridge |
|------|---------|------|-------|----|-------|
| 1 | 08/01–09/27 | 夏〜秋 | 2.15 | 2.16 | *2.64* |
| 2 | 09/28–11/26 | 秋 | 2.13 | 2.32 | 2.52 |
| 3 | 11/27–01/24 | 冬 | 1.96 | 1.84 | 2.15 |
| 4 | 01/25–03/23 | 冬〜春 | **1.56** | 1.65 | **1.61** |
| 5 | 03/24–06/22 | 春〜梅雨 | **1.64** | 1.69 | 1.76 |

- **冬〜春が最も精度高い**（日射変動が小さく予測しやすい）
- **夏〜秋が最も難しい**（台風・にわか雨の変動が大きい）

# まとめ

**このシステムが実現したこと**

1. **データ化**: CSVエクスポートのないスマホアプリから AI で発電データを構築
2. **モデル化**: AMD 農研機構の気象予報と組み合わせて ML モデルを構築
3. **自動化**: 毎朝 6:30 に予測・検証・GitHub push まで全自動
4. **継続検証**: GitHub が毎日更新される検証ダッシュボードとして機能

**精度**: RF v3 MAE = **1.89 kWh/日**（R² = 0.846）

**今後の改善候補**

| 課題 | アプローチ |
|------|-----------|
| 高発電日の過小予測 | Quantile Regression（上側分位点） |
| 夏季精度の改善 | 季節別モデル / 台風フラグ追加 |
| SSD 予報の過小評価 | v3b（SSD なし）との並行運用継続 |
